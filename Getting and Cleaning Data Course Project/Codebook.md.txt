---

## 1. Data Source

**Original Dataset:** Human Activity Recognition Using Smartphones Dataset.  
**Source:** [UCI Machine Learning Repository](http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones).  
**Description:** Data collected from the accelerometer and gyroscope of a Samsung Galaxy S II smartphone worn by 30 subjects while performing six activities: WALKING, WALKING_UPSTAIRS, WALKING_DOWNSTAIRS, SITTING, STANDING, and LAYING.  

---

## 2. Study Design

The dataset is split into **training** (70%) and **test** (30%) sets.  
Each observation corresponds to data from a single subject performing a specific activity, with multiple sensor signal features extracted.  

The project merges training and test sets, filters only variables containing `mean()` or `std()`, replaces activity codes with descriptive names, renames variables for clarity, groups by subject and activity, and calculates the **mean** of each measurement.

---

## 3. Variables in the Final Tidy Dataset

The final tidy dataset **tidy_data.txt** contains:

- **Observations (rows):** 180  
(30 subjects × 6 activities)

- **Variables (columns):** 68  
(2 identifiers + 66 averaged measurement variables)

---

### 3.1 Identifier Variables

1. **subject**  
   - **Description:** ID of the volunteer who performed the activity.  
   - **Type:** Integer.  
   - **Range:** 1–30.  
   - **Units:** N/A.

2. **activity**  
   - **Description:** Activity label performed by the subject.  
   - **Type:** Categorical (Factor).  
   - **Possible values:** WALKING, WALKING_UPSTAIRS, WALKING_DOWNSTAIRS, SITTING, STANDING, LAYING.  
   - **Units:** N/A.

---

### 3.2 Measurement Variables

Each variable corresponds to the **average** of the original measurement across all instances for the given subject and activity.  

- **Domain prefixes**:
  - `Time` → derived from time domain signals.
  - `Frequency` → derived from frequency domain signals (via Fast Fourier Transform).

- **Sensor type**:
  - `Accelerometer` → linear acceleration signal (gravity + body motion).
  - `Gyroscope` → angular velocity signal.

- **Magnitude**:
  - `Magnitude` → Euclidean norm of the 3D signals.

- **Direction suffixes**:
  - `.X`, `.Y`, `.Z` indicate axis direction in 3D space.

- **Statistics**:
  - `Mean` → mean value of measurement.
  - `StandardDeviation` → standard deviation of measurement.

- **Units**:
  - Accelerometer: **g** (standard gravity units).
  - Gyroscope: **rad/s** (radians per second).
  - Magnitudes are also in g or rad/s depending on source.

## 4. Transformations Applied

1. **Merging datasets:** Combined training and test sets into one dataset (`rbind`).
2. **Extracting features:** Selected only variables containing `mean()` and `std()`.
3. **Activity labeling:** Replaced numeric activity codes with descriptive labels using `activity_labels.txt`.
4. **Variable renaming:** Applied clear, descriptive, CamelCase names and expanded abbreviations:
   - `t` → `Time`, `f` → `Frequency`
   - `Acc` → `Accelerometer`, `Gyro` → `Gyroscope`
   - `Mag` → `Magnitude`
   - `mean()` → `Mean`, `std()` → `StandardDeviation`
   - Replace axis suffixes (`-X`, `-Y`, `-Z`) with `.X`, `.Y`, `.Z`
5. **Aggregating:** Grouped by `subject` and `activity` and calculated the **mean** for each variable.
6. **Output:** Saved tidy dataset to `tidy_data.txt` with `write.table(..., row.name=FALSE)`.
